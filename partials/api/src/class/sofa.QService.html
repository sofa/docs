Installation

  • 1. Getting the source

    You can access this class by installing the sofa-q-service package. Installing this package is as easy as running one of the following commands:

    • Via npm:

      npm install sofa-q-service

    • Via Bower:

      bower install sofa-q-service

    • Via Git:

      git clone http://github.com/sofa/sofa-q-service

  • 2. Embedding the source

    After that, embed the distribution file in your HTML document.

    <script src="dist/sofa.qService.js"></script>

    Don't forget that each package defines its own package dependencies. Make sure to embed their files too if needed.

  • 3. Use it

    After that, create an instance of this service by using the `new` operator:

    var qService = new sofa.QService();

  • reject(reason)

    Defined in sofa-q-service/src/sofa.qService.js:176

    Creates a promise that is resolved as rejected with the specified reason. This api should be used to forward rejection in a chain of promises. If you are dealing with the last promise in a promise chain, you don't need to worry about it.

    When comparing deferreds/promises to the familiar behavior of try/catch/throw, think of reject as the throw keyword in JavaScript. This also means that if you "catch" an error via a promise error callback and you want to forward the error to the promise derived from the current promise, you have to "rethrow" the error by returning a rejection constructed via reject.

      promiseB = promiseA.then(function(result) {
    // success: do something and resolve promiseB
    //          with the old or a new result
    return result;
  }, function(reason) {
    // error: handle the error if possible and
    //        resolve promiseB with newPromiseOrValue,
    //        otherwise forward the rejection to promiseB
    if (canHandle(reason)) {
     // handle the error and recover
     return newPromiseOrValue;
    }
    return $q.reject(reason);
  });

    Parameters

    Param Type Details

    Constant, message, exception or an object representing the rejection reason.

    Returns

    Type Details

    Returns a promise that was already resolved as rejected with the reason.
