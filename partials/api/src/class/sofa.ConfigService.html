Installation

  • 1. Getting the source

    You can access this class by installing the sofa-core package. Installing this package is as easy as running one of the following commands:

    • Via npm:

      npm install sofa-core

    • Via Bower:

      bower install sofa-core

    • Via Git:

      git clone http://github.com/sofa/sofa-core

  • 2. Embedding the source

    After that, embed the distribution file in your HTML document.

    <script src="dist/sofa.core.js"></script>

    Don't forget that each package defines its own package dependencies. Make sure to embed their files too if needed.

  • 3. Use it

    After that, create an instance of this service by using the `new` operator:

    var configService = new sofa.ConfigService();

Description

This is a general configuration service which kind of behaves like a registry pattern to make configurations available on all layers. Use this service to access configuration data for you shop.

  • getLocalizedPayPalButtonClass()

    Defined in sofa-core/src/sofa.configService.js:62

    Returns a localized paypal button css class. This method has to be refactored since it currently relies on hard-coded class names. Passing an expression that ends up in a boolean value, tells the method wether to return a class for an enabled or a disabled button.

    For example calling this method like this:

    var className = configService.getLocalizedPayPalButtonClass();

    Returns the class for an enabled button. Calling it with for example true gives us the class for a disabled button.

    var className = configService.getLocalizedPayPalButtonClass(true);

    Returns

    Type Details
    string

    PayPal button class name.

  • get(key, defaultValue)

    Defined in sofa-core/src/sofa.configService.js:92

    Generic getter function that returns a config value by a given key.

    var value = configService.get('foo');

    If a default value is passed and no config setting with the given key exists, it is returned. For example, assuming that no configuration for foo exists, we can default to any custom value like this:

    var value = configService.get('foo', 5); // `5` is a default value

    If no configuration exists and no default value is given, this method returns undefined.

    Parameters

    Param Type Details
    key string

    Key for a certain config value.
    defaultValue object

    A default value which will be returned if given key doesn't exist in config.

    Type Details
